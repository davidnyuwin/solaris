# Solaris v0.11.0-beta

Solaris v0.11.0-beta introduces safe live remote read-only diagnostics execution alongside structured, stateful logging and log filtering features. This release closes the capability gaps of v0.10 by enabling opt-in live probes while strictly preserving the compile-time execution boundaries that protect production targets.

## What's New

### 1. Bracketed IPv6 Connection Support
- Resolves issues where bracketed IPv6 hosts (e.g., `[::1]`) were rejected during port and path parsing. Both IPv4 and bracketed IPv6 formats are now fully supported.

### 2. Runtime Live Remote Policy & User Opt-In Gate
- Live remote operations default to **disabled** (`LiveRemotePolicy.disabled`).
- Toggling "Enable Live Remote Probes" requires explicit user confirmation via an alert dialog.
- Opting out immediately resets the policy and deletes user approval tokens.

### 3. Read-Only Live Probe Runner
- Supports the following four read-only probes:
  - `findHermesBinary` (`which hermes`)
  - `hermesVersion` (`hermes --version`)
  - `hermesStatus` (`hermes status`)
  - `tunnelStatus` (`hermes tunnel-status`)
- Returns a boolean summary only; absolute paths returned by `which` are discarded instantly and never persistent in memory or displayed in the UI.

### 4. Structured Diagnostic Logs
- In-memory diagnostics database mapping logs to `DiagnosticLogEntry` records.
- Records include metadata categorised by source (`app`, `mock`, `localDiagnostics`, `sshPreflight`, `liveProbe`, `redaction`, `privacy`) and severity levels (`info`, `warning`, `error`).

### 5. Stateful Streaming Log Redaction & Filters
- `OutputSanitiser` tracks and flags `isRedacted` based on active regex pattern and path normalisation triggers.
- Streaming holdback suffix buffer holds back partially-transmitted tokens at chunk boundaries (e.g., partial Bearer tokens, Cookie headers, private key blocks, or `/Users/...` paths).
- Dynamic console filters: filter by source, filter by severity, and check "Redacted Only" to review privacy blocks.

---

## Safety Contract & Compile-Time Gates

Solaris v0.11 is built upon a strict, non-negotiable compile-time boundary. 

### Hard Constraints (Never in Release)
The following operations are compiled out of release configurations:
- **Remote Chat / Stdin dispatch**: `#if !DEBUG` guarantees raw prompt submission to SSH stdin remains unavailable.
- **Remote Daemon Restart**: Blocked at compile-time.
- **Tunnel Management (Start / Stop)**: SSH tunneling and network state changes are DEBUG-only.
- **Arbitrary commands**: No shell-wrapping (`sh -c` or `bash`) or free-text SSH entries exist.

---

## Known Limitations

- **Visual Layout Parity**: Visual styling, badges layout, and popover sizing have not been manually verified on a physical target device. Visual inspection is required.
- **Local Unit Testing**: Running `swift test` locally fails on environments without Xcode/XCTest developer toolchains. Authoritative test checking runs automatically on the CI platform.

---

## Upgrade & Installation Notes

1. Configure host, username, port, and identity file paths in **Settings**.
2. Opt-in to remote checks by enabling **Live Remote Probes**.
3. Toggle logs console filters on the **Diagnostics** panel to view specific source events.
