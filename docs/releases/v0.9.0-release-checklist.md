# Solaris v0.9.0 Release Verification Checklist

This checklist guides the developer/QA through verifying a Solaris v0.9.0 build before publishing.

---

## 1. Safety Gates & Compile Constraints
- [ ] **DEBUG Gate Verification:**
  - Verify that `#if DEBUG` macro surrounds the `EnableDeveloperRemoteChat` toggle and the chat stdin execution path.
  - Compile the app in **Release mode** (`swift build -c release`).
  - Verify that the app compiles without exposing any `/chat` command triggers, settings toggles for developer remote chat, or live remote prompt executors.
- [ ] **Default Settings Audit:**
  - Clear `UserDefaults` defaults (or run on a fresh macOS user profile).
  - Verify `EnableDeveloperRemoteChat` defaults to `false`.
  - Verify `UseMockService`/`HermesServiceMode` defaults to Mock Mode.
- [ ] **Secrets & Stdin Check:**
  - Verify that prompt text is sent only through local process `standardInput` pipe data.
  - Verify that the prompt text is **never** appended as arguments to `/usr/bin/ssh` process execution.
  - Verify that no prompt text or LLM responses are written to local log files or system print outputs.
  - Verify that no passwords or private key contents are stored or read.

---

## 2. Input Validation Checks
- [ ] **Host Field Validation:**
  - Input `macmini-cf` (valid alias) -> Verify it is accepted.
  - Input `192.168.1.5` (valid IP) -> Verify it is accepted.
  - Input `host; rm -rf /` (command injection) -> Verify it is rejected.
  - Input `host && echo "hello"` -> Verify it is rejected.
  - Input `host-alias -o ProxyCommand=something` -> Verify it is rejected.
- [ ] **User Field Validation:**
  - Input `sysadmin` -> Verify it is accepted.
  - Input `sysadmin@host` -> Verify it is rejected.
  - Input `sysadmin; rm -rf /` -> Verify it is rejected.
- [ ] **Port Field Validation:**
  - Input `22` -> Verify it is accepted.
  - Input `65535` -> Verify it is accepted.
  - Input `0` -> Verify it is rejected.
  - Input `65536` -> Verify it is rejected.
  - Input `-22` -> Verify it is rejected.
- [ ] **Key File Path Validation:**
  - Choose a valid local private key path (e.g. `~/.ssh/id_ed25519`) -> Verify it is accepted.
  - Clear the path -> Verify it reverts to default SSH agent authentication successfully.
  - Input a nonexistent file path -> Verify it is rejected.
  - Input a directory path (e.g. `/tmp`) -> Verify it is rejected.

---

## 3. SSH Preflight Verification
- [ ] **Connection Test:**
  - Click "Test Remote Connection" on a valid configured host.
  - Verify that Solaris sequentially runs:
    1. `which hermes`
    2. `hermes --version`
    3. `hermes status`
  - Verify that `/chat` is **never** invoked.
  - Verify that successful checks show "Connected. Remote Hermes is reachable."
- [ ] **Error Redaction:**
  - Simulate a connection failure (e.g., set Host to an unreachable local IP).
  - Verify that the UI displays a clean sanitised error (e.g., "Connection timed out" or "Connection refused").
  - Verify that no raw system file paths or remote usernames are visible in the UI error label.

---

## 4. Chat & Output Safety
- [ ] **Mock Chat Execution:**
  - Switch Connection Mode to Mock Mode.
  - Open a session and send a prompt (e.g. "Generate a script").
  - Verify that the output streams incrementally in the chat run card.
  - Verify that markdown layout (headings, bullet points) renders correctly.
  - Verify that code blocks contain language tags and copy controls.
- [ ] **Escape Sequence Safety:**
  - Verify that any raw ANSI color sequences or OSC hyper-links returned in stdout/stderr are stripped or processed cleanly.
  - Verify that no HTML tags are rendered as interactive controls (they must display as literal text).
  - Verify that clipboard copies of responses do not contain hidden links.
- [ ] **Copy Control Verification:**
  - Click copy on a code block -> Verify that the text copied excludes the markdown fences (```) but includes all code text.
  - Click copy on the run card -> Verify that the full plain text response is copied.

---

## 5. Persistence & Lifecycle
- [ ] **Session Navigation:**
  - Verify that creating a new session populates a "New Chat" sidebar row.
  - Verify that switching between sessions loads historical chat runs.
- [ ] **Lifecycle Confirmation Guards:**
  - Trigger "Delete Session" -> Verify that a modal/popover confirms the deletion before removal.
  - Trigger "Clear All History" -> Verify that the modal clearly warns that only local history is cleared.
- [ ] **Active Stream Blockers:**
  - Send a prompt to start streaming.
  - During streaming, try to Rename or Delete the active session.
  - Verify that the session management actions are blocked and a warning banner appears.
- [ ] **Atomic Writes:**
  - Verify that `chat-history.json` is modified atomically via temporary replacement.
  - Verify that only terminal run states (`.completed`, `.failed`, `.cancelled`, `.timedOut`) are persisted to the history file.
