## Release caveat — manual smoke check

The packaged app was launched successfully (`open dist/Solaris.app`) and:
- Process health was confirmed (PID active, no crash reports).
- System log analysis showed normal SwiftUI lifecycle (AppKit state restoration, TextInputUI, CoreSpotlight indexing, no errors).
- Info.plist validated with correct `0.10.0-beta` version string.

**Direct visual UI inspection was limited by macOS `screencapture` permission restrictions.** The following items were NOT visually verified with eyes-on-screen:
- Settings layout correctness
- Mock mode panel rendering
- SSH preflight diagnostic display
- Heartbeat/daemon/tunnel state badges
- Restart control enable/disable states
- Redaction/privacy copy visibility

These are covered by CI tests and code review, but a full visual regression pass should be completed before v0.10 stable.

## Known limitations

1. **Local `swift test` unavailable** — XCTest toolchain limitation on this machine. CI is the authoritative test runner.
2. **IPv6 tunnel hosts unsupported** — Both unbracketed (`::1`) and bracketed (`[::1]`) forms rejected by `isValidRemoteHost()`. Deferred to v0.11.
3. **Remote execution remains mock-only/gated** — `RemoteSSHExecutor` exists but is not wired to the main app flow. No live SSH commands in release builds.
4. **No auto-reconnect loops** — Heartbeat failure must be manually retried.
5. **No remote Hermes discovery** — Remote host must be manually configured.
6. **`actions/checkout@v4` Node 20 annotation** — Cosmetic warning; forced to Node 24 via `FORCE_JAVASCRIPT_ACTIONS_TO_NODE24`.
