# Solaris v0.9.0 Beta Release Notes

## Summary
Solaris v0.9.0 is a developer-gated macOS companion application for the Hermes Agent. This beta release introduces safe session lifecycle management, multi-session navigation, a robust streaming chat terminal view with markdown rendering, copy controls, and configurable connection profiles for remote host execution.

## Major Features

### 1. Connection Profile Configuration & SSH Preflight
* Configurable SSH host/alias, user, port, and identity file path.
* Native AppKit `NSOpenPanel` file picker for selecting SSH private keys securely without reading contents.
* Safe connection preflight checks (`which`, `--version`, `status`) to verify remote Hermes reachability.
* Output sanitisation to clean stderr output before displaying error states in the UI.

### 2. Secure Local Chat Persistence & Session Navigation
* Persistent JSON-based local history file (`chat-history.json`) saved in the user's Application Support directory.
* Dual-pane sidebar navigation supporting multiple chat sessions and run history.
* Safe session rename, delete, and clear-all operations.
* Active stream guards that block destructive session management actions during execution.
* Backward-compatible optional field `isManuallyRenamed: Bool?` to prevent auto-generated titles from overwriting user-specified session names.

### 3. Safe Markdown Rendering & Code Block Ergonomics
* Custom markdown components supporting headings, lists, paragraphs, and inline styles.
* Sanitised language labels and syntax styling (Swift/Shell) for markdown code blocks.
* Interactive copy controls allowing single-click clipboard copying of code block text (excluding fences) and full responses.
* Complete sanitisation of text copied to the clipboard to prevent hidden URL hijacks.

### 4. Direct Stdin Stream Transport
* Prompt text is passed only via stdin, never as part of remote SSH arguments, mitigating shell injection vulnerabilities.
* Process constraints block stdin payloads larger than 16KB.
* Responsive cancellation via the Cancel button, closing pipeline handles and terminating the remote subprocess cleanly.

## Safety Model & Security Boundaries
Solaris v0.9.0 is built on a "local-first, security-first" architecture:
* **Developer Gate:** Live remote chat execution is compiled only under `#if DEBUG` and requires the `EnableDeveloperRemoteChat` setting to be explicitly enabled. It is disabled by default.
* **No Secret Storage:** Solaris does not store private key contents, SSH passwords, or remote tokens.
* **Agent Forwarding Deferral:** SSH Agent Forwarding is not implemented to prevent exposing the local agent socket to compromised remote hosts.
* **Direct Execution:** All subprocesses are launched via local `Process` argument arrays, bypassing local shell interpreters.
* **Output Sanitisation:** Trailing CSI/OSC escape sequence holds and string sanitisation prevent terminal escape injection attacks.

## Known Limitations
* **Passphrase-Protected Keys:** Keys requiring interactive passphrases are not supported because Solaris runs SSH in BatchMode. Users must load keys into `ssh-agent` via `ssh-add` beforehand.
* **App Sandbox:** Arbitrary path resolution for identity file paths is allowed because App Sandbox is disabled. If Sandbox is enabled in future releases, folder permissions or security-scoped URL bookmarks will be required.
