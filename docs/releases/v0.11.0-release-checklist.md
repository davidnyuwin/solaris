# Solaris v0.11.0 — Release Checklist

This gating document lists both the automated and manual checks that must pass before tag and binary distribution of `v0.11.0-beta`.

---

## 1. Automated Validation (CI Gates)

Ensure the latest GitHub Actions CI run for the release candidate commit is green.

- [ ] **Build & Verify Job** passes.
  - Compile Solaris (Debug) passes.
  - Run Unit Tests passes (227 tests run, 0 failures).
  - Run API Smoke Test passes.
  - Run Security & Secret Scan (`./scripts/secret-scan.sh`) passes.
- [ ] **Package & Code Sign Job** passes.
  - App signing verified (`codesign --verify --deep --strict`).
  - Production zip artifact exists (`Solaris-v0.11.0-beta.zip`).
- [ ] **Release Build Compilation** compiles cleanly locally:
  ```bash
  swift build -c release
  ```
- [ ] **Smoke Test Script** executes cleanly:
  ```bash
  ./scripts/smoke-test.sh
  ```
- [ ] **Pre-commit Scan** executes cleanly with zero warnings:
  ```bash
  ./scripts/secret-scan.sh
  ```

## 2. Manual Smoke & Visual Parity (Physical Device)

> [!IMPORTANT]
> The automated pipeline does not perform visual checks. A developer must run the packaged release bundle (`dist/Solaris.app`) on macOS and physically verify the interface.

### UI Layout & Controls
- [ ] **App Launch**: Solaris boots cleanly with no unexpected crashes or terminal windows.
- [ ] **Settings Layout**: Visual elements under Settings -> Remote Host Mode align correctly.
- [ ] **Mock Mode**: Toggling to Mock Mode functions correctly; recent runs and provider lists populate instantly.
- [ ] **Local Diagnostics**: Clicking "Refresh Diagnostics" (Command R) triggers a refresh and populates local process states.

### Safety Opt-in Gates
- [ ] **Policy Disabled State**: Toggle for "Enable Live Remote Probes" starts **off** by default. Setting host or testing connection returns `Live remote checks are disabled` message.
- [ ] **Opt-In Alert Dialog**: Toggling "Enable Live Remote Probes" on displays the approval modal warning.
- [ ] **Cancel Opt-In**: Clicking Cancel inside the modal dismisses the dialog and keeps the toggle disabled.
- [ ] **Approved Opt-In**: Clicking "Enable Read-Only Probes" updates the policy, saves the approved key, and updates status labels.
- [ ] **Disable and Clear**: Toggling the switch off clears approval (`LiveRemotePolicyUserApproved = false`), clears policy, and resets remote status instantly.

### Live Read-Only Probes
- [ ] **Probe Outcome**: Running probes against a valid mock or real remote host updates `findHermesBinary`, `hermesVersion`, `hermesStatus`, and `tunnelStatus` correctly.
- [ ] **Path Redaction**: Full bin paths are ignored and only boolean values display.
- [ ] **Release Blocked Controls**: Chat panel, Daemon Restart button, and Tunnel Start/Stop elements display "This operation is disabled in release builds" or remain locked.

### Diagnostic Logging & Filters
- [ ] **Console Display**: Logs correctly populate in the parsed system log panel.
- [ ] **Source Badges**: Logs carry clear, colored source labels (`APP`, `MOCK`, `LIVEPROBE`, etc.).
- [ ] **Filter Menu**: Selecting a source or severity successfully filters the console output.
- [ ] **Redacted Only**: Toggling the Redacted checkbox displays only the entries that contain redacted data.
- [ ] **Copy/Export Summary**: Copying or exporting visible logs to a text file produces cleanly-formatted, privacy-redacted summaries.

### UI Polish & Aesthetics
- [ ] **Layout Parity**: No visual clipping, truncated text boxes, or overlapping badge elements.
- [ ] **No Hangs/Freezes**: Navigating menus, pausing/resuming log feeds, and switching tabs is fluid and instantaneous.
