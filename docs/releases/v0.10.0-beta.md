# Solaris v0.10.0-beta

**Release date:** 2026-06-04

## Overview

Solaris v0.10 focuses on remote runtime readiness: SSH preflight diagnostics, safe mockable remote command flows, remote state modelling, daemon/tunnel readiness states, and stronger redaction for command diagnostics.

This is a **developer beta** — remote features are modelled and tested behind mock/gated boundaries. No live remote execution is enabled by default.

## What Changed

### SSH Agent / Key Preflight Diagnostics (Phase 1)
- Structured `SSHPreflightDiagnostic` model with pass/warning/fail states.
- `SSHPreflightService` queries `ssh-agent` for loaded keys and detects unreachable agents, empty keychains, missing key files, and encrypted keys requiring passphrase.
- UI diagnostics show actionable guidance (e.g., "Run: ssh-add /path/to/key").
- Key file path validation rejects shell metacharacters and control characters.

### Mock Remote Command Framework (Phase 2)
- `RemoteHermesCommand` enum provides an explicit command allowlist (`which`, `version`, `status`, `chat`, `restart`, `tunnel-start`, `tunnel-stop`, `tunnel-status`).
- `RemoteCommandRunning` protocol defines the execution contract with timeout, stdin, and tunnel request support.
- `MockRemoteCommandRunner` enables deterministic testing without SSH.
- `RemoteCommandBuilder` provides command sanitisation and argument construction with safety guards.

### Remote Connection State Machine & Heartbeat (Phase 3)
- `RemoteConnectionState` enum models the lifecycle: `notConfigured → readyToVerify → verifying → heartbeatPassed/heartbeatFailed`.
- `RemoteConnectionRobustnessState` tracks bounded retry: `stable → degraded → retryAvailable → retrying → retryExhausted → blocked`.
- `RemoteSSHExecutor` runs allowlisted commands over `/usr/bin/ssh` with `BatchMode=yes`, fixed timeouts, and streaming support.
- Heartbeat validation checks remote Hermes binary presence and version compatibility.

### Daemon Monitoring & Safe Restart (Phase 4)
- `RemoteDaemonState` models daemon lifecycle: `unknown → checking → running → stopped → unhealthy → restartAvailable/restartBlocked`.
- Safe restart flow blocks restart when daemon is already restarting or in an unhealthy state.
- Structured test coverage for all state transitions.

### Tunnel Lifecycle & Bounded Retry (Phase 5)
- `RemoteTunnelState` models tunnel lifecycle: `notStarted → preparing → starting → active → degraded → failed → stopped`.
- `RemoteTunnelRequest` validates port ranges (1–65535) and remote host strings.
- Remote host validation rejects colons, whitespace, control characters, and shell metacharacters.
- **IPv6 is not supported in v0.10 by design.** Unbracketed IPv6 literals contain colons that corrupt SSH `-L` arguments. Bracketed IPv6 support is deferred to v0.11.

### Stdin Metadata & Redaction (Phase 6)
- `RemoteCommandInputMetadata` captures audit-safe metadata (command, byte count, timestamp) without storing raw stdin payload.
- `sanitisedFirstLineHint` provides a redacted one-line hint for diagnostics.
- Stdin payload is capped at 16KB.

### Output Sanitisation & Redaction Hardening (Phase 6/6B)
- `OutputSanitiser` provides a 65,536-byte output cap with truncation marker.
- Redacts: API keys, Bearer tokens, Authorization headers, Cookies, GitHub PATs, PEM private keys, secret variable assignments, and **credentialed URLs** (`https://user:pass@host` → `https://[REDACTED_CREDENTIALS]@host`).
- `StreamingOutputSanitiser` provides stateful ANSI/OSC stripping and holdback for partial secret matches.
- `DiagnosticsRedactor` provides privacy-mode redaction for in-app log display.

## Safety Model

- **Command allowlist**: Only enumerated `RemoteHermesCommand` cases may execute. No free-form shell access.
- **No shell interpretation**: SSH is invoked directly (`/usr/bin/ssh`) — never through a shell.
- **BatchMode=yes**: Prevents interactive password prompts.
- **Mock-first**: All remote features operate behind a `MockRemoteCommandRunner` in test and default configurations.
- **Redaction-at-rest**: All command output passes through `OutputSanitiser` before display or logging.
- **Streaming holdback**: Partial secret matches in streaming chunks are held back until safe.
- **Tunnel validation**: Remote host and port are validated before SSH `-L` forwarding.

## Known Limitations

- **IPv6 tunnel hosts are not supported.** Use DNS names or IPv4 addresses.
- **Remote execution is mock-only.** Live SSH execution is gated and not enabled by default.
- **No auto-reconnect loops.** Heartbeat failure must be manually retried.
- **No remote Hermes discovery.** Remote host must be manually configured.
- **Local `swift test` unavailable** due to XCTest toolchain limitations. CI is the authoritative test runner.
- **Diagnostic log display** uses `LazyVStack`/`ScrollView` — very large logs may affect scroll performance (core sanitiser cap at 65,536 bytes mitigates this).

## What Is Intentionally Still Mock/Gated

- `RemoteSSHExecutor` is available but not wired to the main app flow by default.
- Tunnel start/stop is modelled but requires manual configuration.
- Chat over SSH (`hermes chat -q - -Q`) is in the allowlist but not exposed in the UI.
- Daemon restart requires explicit user action and is blocked in certain states.
