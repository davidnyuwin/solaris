## Prerequisites

All items must be completed before any Phase 3 implementation code is written.

### Architecture Contract

- [ ] **ADR-001 safety contract reviewed** — all sections read and understood.
- [ ] **Release allowlist accepted** — `whichHermes`, `hermesVersion`, `hermesStatus`, `gatewayStatus`, `tunnelStatus` approved for release with user opt-in.
- [ ] **DEBUG-only operations accepted** — `hermesRestart`, `hermesChat`, `tunnelStart`, `tunnelStop` approved for DEBUG builds only.
- [ ] **Forbidden operations accepted** — arbitrary SSH, shell strings, filesystem browsing, env dumping, config scraping, package installs, service modification all confirmed forbidden.
- [ ] **Non-goals acknowledged** — 12 non-goals in ADR-001 Section 12 accepted.

### Policy Design

- [ ] **`LiveRemotePolicy` enum design accepted** — `.disabled`, `.readOnly`, `.full` cases.
- [ ] **Default policy is `.disabled`** — fresh install has live remote off.
- [ ] **Policy enforcement path defined** — compile-time `#if !DEBUG` + runtime policy check.
- [ ] **`.full` never reachable in release** — compile-time gate confirmed.

### User Approval

- [ ] **UI copy approved** — "Enable Live Remote Probes?" dialog copy.
- [ ] **Blocked operation copy approved** — "This operation is disabled in release builds."
- [ ] **Per-probe confirmation design accepted** — one-time confirmation on first probe.
- [ ] **UI labels do not suggest SSH terminal capability** — "Remote Hermes Status", not "Remote SSH Terminal".

### Kill Switch

- [ ] **Runtime kill switch defined** — `AppStorage("LiveRemotePolicy")` set to `.disabled`.
- [ ] **Kill switch takes effect immediately** — no cached connections survive.
- [ ] **Compile-time reinforcement confirmed** — `#if !DEBUG` remains as ultimate guard.
- [ ] **Rollback strategy defined** — default `.disabled` in next release if bug found.

### Logging and Redaction

- [ ] **Logged data accepted** — command type, timestamp, duration, exit code, stdout/stderr length only.
- [ ] **Not-logged data accepted** — raw stdin, key material, passphrases, bearer tokens, raw stdout/stderr.
- [ ] **Redaction path accepted** — all output through `OutputSanitiser` before display or storage.
- [ ] **Output bounds accepted** — 100KB hard cap, 16KB stdin cap, 4096 streaming chunks.

### Failure States

- [ ] **Failure state machine accepted** — all `RemoteConnectionState` cases mapped to recovery actions.
- [ ] **Retry policy accepted** — max 3 retries with exponential backoff for probes.
- [ ] **Timeout policy accepted** — 8s probes, 5s SSH connect, 30s DEBUG-only operations.
- [ ] **Sanitised failure messages accepted** — static strings only, no dynamic remote content.

## Rollback Plan

If a live remote execution bug is discovered after release:

1. **Immediate:** Ship a point release with `LiveRemotePolicy` defaulting to `.disabled`.
2. **Short-term:** Add additional guards around the specific bug.
3. **Long-term:** Consider a server-side kill switch (URL check for disable flag).
4. **Always:** The `#if !DEBUG` compile-time gate remains and cannot be bypassed by configuration.
