# Solaris v0.9 — Batch 2M: Connection Profile Inputs and SSH Preflight Validation

This document establishes the design, security boundaries, validation rules, and preflight behavior for managing remote host SSH connection profiles in Solaris v0.9.

---

## 1. Connection Profile Fields

Solaris allows users to configure a custom SSH connection profile in the "Remote Host Mode" settings section. The fields map to non-secret connection parameters stored in `UserDefaults`:

1. **Host:** SSH alias (e.g., `macmini-cf`), hostname, or IP address.
2. **User (Username):** Optional SSH login username.
3. **Port:** Optional SSH custom port. Defaults to `22` if unspecified.
4. **Key File (Identity File):** Optional path to an SSH private key file on the local machine.

---

## 2. Validation & Safety Rules

To prevent command injection, directory traversal, and credential leakage, strict validation checks are performed on settings inputs before any process execution.

### Host Validation
* **Rule:** Must not be empty, must not contain spaces, and must not contain shell metacharacters.
* **Allowed Characters:** Alphanumerics, `.`, `-`, `_`.
* **Security boundary:** Prevents embedding arbitrary SSH arguments (e.g. `-o ProxyCommand=...`) or shell commands (e.g. `; rm -rf /`) in the host field.

### Username Validation
* **Rule:** Optional. If provided, must not contain spaces, `@`, or shell metacharacters.
* **Allowed Characters:** Alphanumerics, `-`, `_`.
* **Security boundary:** Rejects format like `user@host` if host is stored separately, preventing metacharacter execution.

### Port Validation
* **Rule:** Optional. If provided, must be an integer between `1` and `65535`.
* **Allowed Values:** `1...65535`.
* **Security boundary:** Prevents negative values, decimals, letters, and ports outside the standard range.

### Identity File (Key File) Validation
* **Rule:** Optional. Path must not contain control characters.
* **Existence check:** If specified, the file must exist at the local path and must not be a directory.
* **Redaction rule:** Solaris only stores the path, never the private key contents. The UI displays the key file's basename or a redacted path to protect absolute user directories.

---

## 3. SSH Argument Construction Safety

To eliminate shell execution risks, Solaris maps settings directly to safe process arguments:

* **No Shell Parsing:** Commands are executed directly via AppKit's `Process` (POSIX `execve`), avoiding any giant local shell command strings or shell interpretation.
* **Argument Splitting:** Host, custom ports (`-p`), and identity files (`-i`) are passed as separate, distinct elements in the process arguments array (`[String]`).
* **Example argument mapping:**
  ```swift
  ["-p", "2222", "-i", "/Users/sysadmin/.ssh/id_ed25519", "-o", "BatchMode=yes", "sysadmin@macmini-cf", "which", "hermes"]
  ```

---

## 4. SSH Agent Forwarding Policy

Agent forwarding expands the trust boundary because a compromised remote host can gain access to the local user's forwarded agent socket.

* **Policy:** SSH Agent Forwarding is **not implemented** in Solaris.
* **Toggle placeholder:** The UI explicitly states that Agent Forwarding is deferred pending a separate security review.
* **Notice copy:**
  > Agent forwarding is not implemented. It is intentionally deferred pending a separate security review.

---

## 5. SSH Preflight Validation Flow

The "Test Connection" action verifies the connection status by sequentially running a series of three safe allowlisted command checks:

1. **`which hermes`**: Finds the Hermes executable path on the remote host.
2. **`hermes --version`**: Fetches the remote Hermes version.
3. **`hermes status`**: Checks if the remote Hermes service is active.

### Security Guarantees
* **Fixed Allowlist:** Solaris only executes fixed, pre-compiled commands on the remote machine. No arbitrary remote command execution is allowed.
* **No Chat Execution:** The `/chat` command is **never** used during preflight validation.
* **Error Sanitisation:** To prevent exposing raw filepaths, usernames, or hostnames in the UI, stderr outputs are cleaned by `sanitiseSSHError(_:)` before display.
* **Subprocess Timeout:** All SSH processes enforce a strict connection and command execution timeout (default 8s), terminating the subprocess if unresponsive.

---

## 6. Documented Boundaries

* **No SSH Config Editing:** Solaris **does not** edit, read, or modify `~/.ssh/config`.
* **No Private Key Storage:** Solaris **does not** read or persist the contents of SSH private keys.
* **No SSH Key Creation:** Solaris **does not** create or modify local SSH keys.
* **No Cloudflare Tunnel Alteration:** Solaris **does not** configure or manage Cloudflare Tunnels.

---

## 7. Remaining Blockers

* **Key File Sandbox Entitlements:** If App Sandbox is enabled in future releases, path resolution of arbitrary identity keys outside the sandbox container will require security-scoped bookmarks or user-initiated folder permissions. Currently, sandbox is disabled.
* **Passphrase-Protected Keys:** Keys requiring a passphrase are not interactive since Solaris runs SSH in `BatchMode=yes`. Users must load passphrase-protected keys into their macOS SSH agent (`ssh-add`) prior to connecting.
