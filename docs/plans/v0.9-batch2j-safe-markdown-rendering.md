# Solaris v0.9 — Batch 2J: Safe Markdown Rendering & Chat Bubble Styling

This document establishes the safe markdown parsing rules, HTML/link/image sanitisation policies, and conversational chat bubble layouts implemented in Solaris v0.9 Batch 2J.

---

## 1. Safe Markdown Rendering Strategy

Solaris adopts a **sandboxed SwiftUI-native block rendering** strategy. Model responses are untrusted and must never be evaluated in dynamic runtimes (like `WKWebView` or JavaScript contexts). Instead, raw strings are parsed line-by-line in Swift and mapped directly into isolated SwiftUI layout primitives.

### Rendering Pipeline:
```
  [ Raw Streaming Response ]
             │
             ▼
      OutputSanitiser      (Global ANSI stripping, credential redacting)
             │
             ▼
       parseMarkdown       (Splits lines statefully into safe semantic blocks)
             │
             ▼
     prepareSafeInline     (Strips raw images, formats links, shields raw HTML)
             │
             ▼
   MarkdownMessageView     (SwiftUI-native block styling and text concatenation)
```

---

## 2. Supported Markdown Subset

Solaris parses and supports the following clean Markdown blocks:

*   **Headings:** Lines starting with `#` to `######`. Rendered as white text with scale-down font sizes (H1 = 18pt bold, H2 = 16pt bold, etc.).
*   **Fenced Code Blocks:** Content enclosed in ` ``` ` sequences. Rendered inside a background block using monospaced font, with horizontal scrolling.
*   **Bullet Lists:** Lines starting with `* `, `- `, or `+ `. Rendered as indented bullet blocks.
*   **Numbered Lists:** Lines starting with numbers followed by dot and space (e.g. `1. `). Rendered as numbered blocks.
*   **Blockquotes:** Lines starting with `> `. Indented, styled with italic font, and bordered with a `hermesPurple` brand bar.
*   **Paragraphs:** Clean inline styling (bold `**text**`, italic `*text*`, and monospaced inline code `` `code` ``) is supported via text concatenation (`+`).

---

## 3. Strict Safety & Content Policies

### Raw HTML Policy
*   Raw HTML tags are not parsed. Tag boundaries like `<script>` or `<div>` are escaped/treated as literal plain text by the SwiftUI `Text` renderer, completely preventing injection attacks or script evaluation.
*   No `WebView` or HTML renderers are used.

### Image Policy
*   Markdown image tags (`![alt](url)`) are blocked before rendering.
*   The pre-processor replaces all image syntax with a static text placeholder: `[image omitted]`. No remote requests are ever sent.

### Link Policy
*   Standard Markdown links (`[anchor](url)`) are converted to plain text displaying the `anchor` description only.
*   Links are not clickable, do not auto-open, do not show previews, and never load remote metadata.

### Code Block Safety
*   Code blocks preserve formatting, whitespace, and indentation.
*   No dynamic parsing or script execution is allowed. No syntax highlighting libraries are imported to avoid adding external parser vulnerabilities.

### Malformed Markdown Fallback
*   If block parsing fails or produces an empty list from non-empty inputs, Solaris falls back to rendering the clean plain text to ensure responses are never lost.

---

## 4. Chat Bubble Layout and Styling

To present conversational outputs cleanly:

*   **User Prompt Bubble:** Styled as a distinct right-aligned capsule with a dark background (`Color.white.opacity(0.06)`) and a green `person.fill` icon. Defaults to `Prompt not saved` if the prompt is missing.
*   **Assistant Response Bubble:** Left-aligned, styled with a distinct black background (`Color.black.opacity(0.12)`). Contains the `MarkdownMessageView` for markdown content.
*   **Status & Error Indicators:** Failed, timed-out, or cancelled chat runs display visible, colored status tags (`Failed`, `Timeout`, `Cancelled`) and apply matching border highlights to the assistant bubble (e.g. red for cancellation, amber for timeout).
*   **Streaming Continuity:** Incremental text chunks flow through the stateful sanitiser and render progressively without layout jitter.

---

## 5. Remaining Blockers

*   **Syntax Highlighting:** While code blocks scroll and preserve spacing, color-coded syntax highlights are deferred.
*   **Interactive Controls:** Copy-to-clipboard shortcut buttons next to code blocks are currently deferred.
*   **Multi-Session Management:** Sidebar navigation is operational, but manual session deletion, renaming, or archiving remains deferred.
