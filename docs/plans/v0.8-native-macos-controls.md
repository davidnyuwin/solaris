# Solaris v0.8 — Native macOS Controls

## Status

**In Progress:** Batch 3 (Pause/Resume Logs Shortcut) implemented in `v0.8.0-dev`.

---

## Goal

Add native macOS keyboard and menu controls for safe diagnostics actions, starting with manual refresh.
No new Hermes integration is added. All commands call only the existing read-only diagnostics refresh path.

---

## Non-Goals

*   No new Hermes CLI commands.
*   No WebSocket.
*   No command sending.
*   No Local Profiles Discovery.
*   No reading `~/.hermes/config.yaml`.
*   No mutating Hermes commands.
*   No new data surfaces.

---

## Batch 1: Diagnostics ⌘R Keyboard Shortcut

### Implemented

*   [x] `FocusedValues+Solaris.swift` — defines `hermesViewModel` and `navigationSelection` focused value keys.
*   [x] `DiagnosticsCommands.swift` — `Commands` group adding a native "Diagnostics" menu with:
    *   "Refresh Diagnostics" menu item (⌘R).
    *   Disabled when Local Diagnostics is not the active view.
    *   Disabled when a refresh is already in progress.
    *   Calls only the existing safe `refreshDiagnostics()` path.
*   [x] `HermesCompanionApp.swift` — wires `DiagnosticsCommands` into the scene via `.commands {}`.
*   [x] `MainView.swift` — publishes `hermesViewModel` and `navigationSelection` as focused values from the detail pane.
*   [x] `ProvidersView.swift` — refresh button label updated to show `⌘R` hint. Accessibility hint updated to mention the keyboard shortcut.

### Behaviour

| Situation | ⌘R behaviour |
|---|---|
| Local Diagnostics active, idle | Triggers `refreshDiagnostics()` |
| Local Diagnostics active, already refreshing | Menu item disabled — no-op |
| Any other view active | Menu item disabled — no-op |

## Batch 2: Export Redacted Diagnostics

### Implemented

*   [x] `HermesViewModel.makeRedactedDiagnosticsSummary()` — reusable redacted summary generator (replaces `generateDiagnosticsSummary()`).
*   [x] `HermesViewModel.exportRedactedDiagnosticsSummary()` — NSSavePanel-based export that never writes automatically.
*   [x] `DiagnosticsCommands.swift` — "Export Redacted Diagnostics…" menu item (⌘⇧E), disabled on non-diagnostics views.
*   [x] `ProvidersView.swift` — compact "Export…" button in the logs console toolbar, with feedback states ("Exported", "Export cancelled", "Export failed").
*   [x] Always uses `DiagnosticsRedactor` to redact local paths, PIDs, and token-like strings.
*   [x] Suggested filename: `Solaris-Diagnostics-YYYY-MM-DD-HHMM.txt`.
*   [x] `.txt` type restriction via Uniform Type Identifiers.

### Behaviour

| Situation | Behaviour |
|---|---|
| Local Diagnostics active, user selects Export | NSSavePanel opens with suggested filename |
| User clicks Export in save panel | Redacted summary written to selected path |
| User cancels save panel | `exportFeedbackText = "Export cancelled"` — not an error |
| Write fails (permissions, disk full) | `exportFeedbackText = "Export failed"` |

## Batch 3: Pause/Resume Logs Shortcut

### Implemented

*   [x] `DiagnosticsCommands.swift` — "Pause Diagnostics Logs" / "Resume Diagnostics Logs" menu item (⌘⇧P), disabled on non-diagnostics views.
*   [x] Dynamic label that changes based on `isDiagnosticsLogPaused` state.
*   [x] Calls the existing safe `toggleDiagnosticsLogPause()` — no new state, no CLI calls, no disk writes.
*   [x] `ProvidersView.swift` — button label updated to show `⌘⇧P` hint. Accessibility hint updated.
*   [x] Uses `Command + Shift + P` to avoid conflict with the standard macOS File > Print shortcut (⌘P).

### Behaviour

| Situation | ⌘⇧P behaviour |
|---|---|
| Local Diagnostics active, logs live | Menu item: "Pause Diagnostics Logs" — pauses display |
| Local Diagnostics active, logs paused | Menu item: "Resume Diagnostics Logs" — resumes display |
| Any other view active | Menu item disabled — no-op |
