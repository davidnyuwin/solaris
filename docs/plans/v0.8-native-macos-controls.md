# Solaris v0.8 — Native macOS Controls

## Status

**Complete:** All 5 batches implemented in `v0.8.0-dev`.

---

## Goal

Add native macOS keyboard and menu controls for safe diagnostics actions, starting with manual refresh.
No new Hermes integration is added. All commands call only the existing read-only diagnostics refresh path.

---

## Non-Goals

*   No new Hermes CLI commands.
*   No WebSocket.
*   No command sending.
*   No Local Profiles Discovery.
*   No reading `~/.hermes/config.yaml`.
*   No mutating Hermes commands.
*   No new data surfaces.

## Batch 4: CLI Status Failure Presentation

### Implemented

*   [x] `CLIStatusPresentation.swift` — `CLIStatusKind` enum with seven states: available, timedOut, pythonMissing, nonZeroExit, emptyStdout, noFieldsParsed, unknown. Each carries a short `label` and user-facing `explanation`.
*   [x] `classifyCLIStatus()` free function that maps raw CLI status strings (produced by `LocalHermesDiagnosticsService`) to `CLIStatusKind`.
*   [x] `ProvidersView.swift` — CLI status row now uses `classifyCLIStatus()` for accurate color mapping:
    *   Available → green (`.stable`)
    *   Timed out → amber (`.idle`)
    *   Python missing → rose (`.unavailable`)
    *   Non-zero exit → rose (`.unavailable`)
    *   Empty stdout → amber (`.idle`)
    *   No fields parsed → amber (`.idle`)
*   [x] Row name updated to "Read-only CLI Status" with per-state explanation in detail text.
*   [x] `ProcessStatusRow.swift` — added optional `customAccessibilityHint` parameter so CLI status row can provide specific guidance.
*   [x] CLI status row accessibility hint: "Shows whether Solaris can read safe Hermes status information."
*   [x] Tests for all seven `classifyCLIStatus` states.

### Colour / State Mapping

| Raw CLI Status | Badge | Colour | Behaviour |
|---|---|---|---|
| Available | Stable | Green | CLI checks working |
| Timed out | Idle | Amber | Check timed out |
| Unavailable (Python missing) | Unavailable | Rose | Python not found |
| Unavailable (Exit code: N) | Unavailable | Rose | CLI returned error |
| Parse warning (Empty stdout) | Idle | Amber | No CLI output |
| Parse warning (No fields parsed) | Idle | Amber | Unknown CLI output format |
| nil / other | Unavailable | Rose | Not available |

## Batch 5: Accessibility Announcements

### Implemented

*   [x] `AccessibilityAnnouncer` — a small utility that posts macOS VoiceOver announcements via `NSAccessibility.post(element:notification:userInfo:)` with the `AXAnnouncement` key.
*   [x] `DiagnosticsRefreshSource` enum (`.manual`, `.scheduled`) added to `HermesViewModel`.
*   [x] `refreshDiagnostics(source:)` uses the source to decide whether to announce:
    *   **Manual refresh success:** announces "Diagnostics refreshed."
    *   **Manual refresh failure:** announces "Diagnostics refresh failed."
    *   **Scheduled refresh success:** silent (no announcement).
    *   **Scheduled refresh failure:** announces "Scheduled diagnostics refresh failed." (throttled to 60 s cooldown).
*   [x] Copy success: announces "Diagnostics summary copied."
*   [x] Export success: announces "Diagnostics summary exported."
*   [x] Scheduler passes `.scheduled` source; manual button and menu command default to `.manual`.

### Anti-spam Behaviour

| Situation | Announcement | Throttled |
|---|---|---|
| Manual refresh success | "Diagnostics refreshed." | No |
| Manual refresh failure | "Diagnostics refresh failed." | No |
| Scheduled refresh success | (silent) | N/A |
| Scheduled refresh failure | "Scheduled diagnostics refresh failed." | Yes, 60 s cooldown |
| Copy success | "Diagnostics summary copied." | No |
| Export success | "Diagnostics summary exported." | No |

### Accessibility API

*   Uses `NSAccessibility.post(element:notification:userInfo:)` with the announcement text key `AXAnnouncement`.
*   Gate: checks `NSWorkspace.shared.isVoiceOverEnabled` first — no-op when VoiceOver is off.
*   No private APIs, no dependencies, no new frameworks.
