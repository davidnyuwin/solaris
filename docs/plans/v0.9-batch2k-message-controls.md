# Solaris v0.9 — Batch 2K: Message Controls & Code Block Ergonomics

This document establishes the plan, implementation details, clipboard safety policies, and ergonomics improvements for code blocks and message bubbles implemented in Solaris v0.9 Batch 2K.

---

## 1. Ergonomic Message Controls

To make Solaris more usable for development without introducing security/execution risks, we introduce explicit clipboard copy controls on three levels:

1. **Code Blocks:** A copy button placed in the header of each rendered code block allows copying only the code content, excluding the markdown triple backtick fences or language identifiers.
2. **Assistant Response Bubbles:** A "Copy Response" button placed near the footer of the assistant response bubble allows copying the entire rendered response cleanly.
3. **User Prompt Preview Bubbles:** A copy button placed next to the user bubble (where prompt previews exist) allows copying the prompt.

---

## 2. Clipboard Safety Rules

Copying content must never leak unsanitised credentials, ANSI escape sequences, or hidden URLs. The following strict rules are applied to all clipboard copy actions:

* **Plain Text Only:** The clipboard operations write strictly plain text (`NSPasteboard.PasteboardType.string`). Rich HTML, RTF, attributed text, and images are completely forbidden.
* **No Raw Content Copying:** Clipboard actions copy from the already-sanitised view representation. Raw stdout, raw stderr, raw API responses, or raw/unfiltered prompt inputs are never directly copied.
* **Link Target Stripping:** Under Markdown links (`[anchor](url)`), the `url` target is stripped before copying. Only the user-visible `anchor` text is copied to prevent silent exfiltration or redirection via hidden URLs.
* **Image URL Stripping:** Markdown image targets (`![alt](url)`) are replaced with the plain-text string `[image omitted]` before writing to the pasteboard.
* **HTML Tag Escaping:** Raw HTML tag syntax is treated as literal plain text; no HTML rendering or dynamic script parsing is supported.
* **Fence Stripping:** Markdown backtick code fences (e.g. ` ```swift ` and ` ``` `) are stripped from the response text to ensure clean copy output.

---

## 3. Code Block Ergonomics & Language Labels

Solaris parses and processes fenced code block metadata to improve formatting:

* **Language Identifier Extraction:** If a code fence provides an optional language label (e.g. ` ```swift `), the label is extracted, sanitised, and displayed as a clean tag at the top left of the code block.
* **Strict Label Validation:** To prevent injection or layout disruption:
  * Only alphanumeric characters and `+`, `-`, `#`, `_` are allowed in the label.
  * Spacing, tabs, whitespace, control characters, or symbols (like `!`, `@`, `$`, etc.) trigger immediate fallback: the label is rejected (reverts to `nil`) and defaults to `code`.
  * The language label is capped at a maximum of 24 characters.
* **Horizontal Scrolling:** Monospaced code blocks support horizontal scrolling via SwiftUI `ScrollView` to maintain stable bounds.

---

## 4. Lightweight Syntax Styling Decision

To keep the codebase lightweight and maintainable without external library dependencies (which would increase security surface area or compile times):

* **Keyword Coloring:** Solaris includes a lightweight, safe keyword-coloring helper (`highlightCode`) for the two primary languages used in companion interactions: **Swift** and **Shell/Bash**.
* **Safe Parser:** Keywords are matched using a simple Scanner and colored using native SwiftUI colors (e.g., `Color.hermesPurple` for Swift keywords, `Color.amber` for Shell keywords, and `.hermesTeal` as base).
* **Failure-safe Fallback:** Any parsing issue or unsupported language gracefully falls back to plain monospaced text in the default brand color (`Color.hermesTeal`).
* **Intentional Deferral of Full Syntax Engine:** Implementing a full syntax highlighting engine (like Pygments or Highlight.js) or a WebView-based renderer is intentionally deferred to maintain strict sandboxing and binary size targets.

---

## 5. Remaining Blockers

* **Live SSH remote chat gate:** Remains disabled by default to prevent unintentional command runs on remote hosts.
* **Manual Session Archival:** Session lifecycle operations (delete, rename, archive) remain deferred.
